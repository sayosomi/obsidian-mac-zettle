## 背景

  Android の Termux から本プロジェクト API を日常利用するため、モバイル入力に最適化した対話式 CLI を実装する。既存
  API（Bearer 認証、/prototypes など）を前提とし、最小打鍵で安全に操作できる UX を提供する。

## 目的

  - Termux 上で `プロトタイプ作成` を対話形式で完結できること
  - 入力負荷を最小化すること（HHMM、曜日連続入力、1項目ずつ編集）
  - 確認画面で部分修正を繰り返せること（番号指定編集）
  - 将来的にホームメニューから他エンドポイント操作（daily-tasks/sessions/off-days など）を段階的に追加しやすい構造に
  すること

## 対象スコープ（初期）

  - ホーム画面
  - プロトタイプ新規作成フロー
  - 確認画面ループ
  - タグ/URI の複数入力 + 項目内確認 + 番号指定修正/削除
  - 頻度サブメニュー（daily/weekday/monthly_day/yearly_date/monthly_nth_weekday/last_execution_boundary）
  - 頻度配下の offDayBehavior 設定
  - 作成完了後の遷移（ホームへ戻る / 今日のタスク追加 / 継続作成）

## 非スコープ（初期）

  - ホームのダミー機能の本実装
  - 既存 API の仕様変更
  - GUI 化

## 拡張性方針（明記）

  - ホームメニューは「機能ごとの独立ハンドラ」を呼び出す構成にする
  - API 呼び出しは共通クライアント層に集約し、認証・エラー整形を統一する
  - 画面文言はテンプレート化し、機能追加時も既存フローに影響を出しにくくする
  - 入力/検証ロジックを共通部品化し、他エンドポイント実装時に再利用可能にする

## 成功条件

  - `name` のみで最短作成できる
  - すべての編集項目が確認画面から再編集可能
  - API 送信 payload が仕様に一致する
  - エラー時にユーザーが復帰可能（再入力・戻る・終了）
  - 新規メニュー項目（別エンドポイント）を既存コードの最小変更で追加できる
