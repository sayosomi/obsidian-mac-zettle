CLIの[ホーム]画面の選択肢に`セッション`を追加する
1キーで画面遷移する
使用エンドポイント
GET /sessions/overview 
POST /sessions/start
POST /sessions/stop
startAtとstopAtは省略し、サーバータイムを使う。クライアントとの軽微なズレは許容する。
経過時間と残り時間はクライアント側で計算し、1秒毎に経過時間と残り時間の表示を更新したいんだけど、できる？デメリットはある？教えて
## 実行中セッションがある場合

```
[セッション > 開始]
2026-03-10（火）14:32 WORK
名前: task
開始: 13:20:00
経過:  1h 12m  1s
残り: 10h  0m 59s
#focus #spec
https://example.com/task
https://example.com/hoge
Enter) サマリー表示
p) プロトタイプ詳細
e) 開始時刻を編集
b) 開始前に戻る
h) ホームへ戻る
q) 終了
```
タグとURLはmuted color
Enterで[セッション > 開始 > 停止]へ
pで[セッション > 開始 > 編集]へ

```
[セッション > 開始 > 編集]
2026-03-10（火）14:32 WORK
名前: task
開始: 13:20:00
p) プロトタイプ詳細
b) 開始画面に戻る
h) ホームへ戻る
q) 終了
```

```
[セッション > 開始 > 停止]
2026-03-10（火）14:32 WORK
名前: task  
開始: 13:20:00  
停止: 14:32:01
経過: 1h 12m 1s  
残り: 10h 0m 59s
Enter) サーバーに送信
p) プロトタイプ詳細
e) 停止時刻を編集
b) 停止前へ戻る
h) ホームへ戻る
q) 終了
```
`番号入力で編集`はmuted color
`Enter) サーバーに送信` はwarning color
