CLIの[ホーム]画面の選択肢に`セッション`を追加する
1キーで画面遷移する
使用エンドポイント
GET /sessions/overview 
POST /sessions/start
POST /sessions/stop
startAtとstopAtは省略し、サーバータイムを使う。クライアントとの軽微なズレは許容する。
経過時間と残り時間はクライアント側で計算し、1秒毎に経過時間と残り時間の表示を更新したいんだけど、できる？デメリットはある？教えて
## 実行中セッションがある場合

```
[セッション]
2026-03-10（火）14:32 WORK
名前: task
開始: 13:20:00
経過:  1h 12m  1s
残り: 10h  0m 59s
#focus #spec
https://example.com/task
https://example.com/hoge
Enter) サマリー表示
p) プロトタイプ詳細
h) ホームへ戻る
q) 終了
```
タグとURLはmuted color
Enterで[セッション >  サマリー]へ
```
[セッション > サマリー]
2026-03-10（火）14:32 WORK  
名前: task  
1) 開始: 13:20:00  
2) 停止: 14:32:01
経過: 1h 12m 1s  
残り: 10h 0m 59s
番号入力で編集
Enter) サーバーに送信
p) プロトタイプ詳細
b) 実行中画面へ戻る
h) ホームへ戻る
q) 終了
```
`番号入力で編集`はmuted color

```
[セッション > サマリー > 編集（停止時刻）]
  ストップを記録しました。

  対象日: 2026-03-10（火）
  stopAt: 14:45:00
  名前: task
  実績: 00:25:00

  1) ホームへ戻る
  2) プロトタイプ詳細
  q) 終了
```
